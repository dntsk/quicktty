## Результат

Подписанная alpha GhostTerm готова и установлена пользователем. Пользователь сообщил, что пишет из установленного приложения и основные функции работают.

`.build/` игнорируется Git; DMG и evidence не входят в commit.

## Реализовано

- Версия bundle: `0.1.0`, build `1`; alpha label используется в имени DMG.
- Добавлена утверждённая временная иконка GT и воспроизводимый AppKit generator.
- В app bundle добавлены Ghostty terminfo, shell integration, встроенные terminal themes и license notice.
- Release script выполняет arm64 Xcode archive, Developer ID signing, hardened runtime validation и создаёт подписанный DMG.
- Notarization script использует только Keychain profile, проверяет ответ Apple, stapler, codesign и Gatekeeper.
- Release input paths, generated resource replacement, signal cleanup, stale evidence и partial artifacts защищены.
- Make release targets отклоняют опасные command-line variable expansions; release variables нужно передавать через process environment перед `make`.
- Исправлены два теста: устаревший `Cmd+B` заменён на свободный Control+G, а process-liveness fixture использует long-lived `sleep` вместо `cat`.
- Исправлена проверка финального SHA-256 notarized DMG: ровно 64 hex-символа и production-call contract.

## Важный контекст

- Notary profile: `ghostterm-notary` в login Keychain; credentials не находятся в репозитории.
- Team ID и identity передаются через environment, не зашиты в scripts.
- Запущенное установленное приложение не завершать и не заменять без явного запроса пользователя.
- Изменения после создания DMG затрагивали только Make/tests/notarization digest contract; production app code и содержимое выпущенного bundle не менялись, повторная notarization не требуется.
