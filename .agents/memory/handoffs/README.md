# Передача контекста между сессиями

При завершении рабочей сессии создавайте здесь Markdown-файл с именем:

`YYYY-MM-DD-HHMM-краткая-тема.md`

Не перезаписывайте предыдущие handoff. Документ должен быть кратким и содержать только проверяемый контекст для продолжения работы.

## Шаблон

```markdown
# Handoff: <тема>

- **Дата:** YYYY-MM-DD
- **Ветка:** <ветка>
- **Статус дерева:** <чистое / есть незакоммиченные изменения>

## Выполнено

- <завершённая работа и ключевые файлы>

## Проверки

- `<команда>` — <результат>

## Незавершённое

- <что осталось и почему>

## Следующий шаг

1. <конкретное первое действие>

## Важный контекст

- <инварианты, риски, принятые решения>
```

Не включайте секреты, содержимое `.env`, ключи, токены и персональные данные. Устойчивые архитектурные решения переносите также в `../architecture-decisions.md`, изменения внешних контрактов — в `../integration-contracts.md`, завершённые задачи — в `../tasks-completed.md`.
